# AINative OS 架构总览

## 1. 目标

在应用层构建一个 AINative 操作系统：

1. 复用文件系统范式组织 `Topic` 与 `Node`。
2. 以 Agent 作为可调度的执行资源；若需要不同工作风格，优先在能力层通过 Prompt / Plan 组装，而不是在资源层固化角色。
3. 把调度、执行、能力注入、模型调用解耦，支持并行开发与演进。

## 2. 核心设计决策

1. 任务中心而非 CPU 中心：系统重点控制任务，不强绑定 Agent 与队列。
2. 双队列调度：一级按主题，二级按叶子 Node 优先级。
3. 交互驱动优先：一级优先级由用户交互热度决定，采用多级反馈思想。
4. 可插拔能力层：上下文、工具、skill、存储抽象化，避免过早绑定实现。

## 3. 模块拆分

1. 虚拟文件系统：`Node/Topic` 抽象、树结构组织、叶子节点可操作约束。
2. 调度队列：双队列、反馈调度、抢占与并发分配。
3. Agent 池：资源分配、限流、健康管理、原始计量。
4. Agent 能力：上下文注入、工具注入、skill 注入、对话存储。
