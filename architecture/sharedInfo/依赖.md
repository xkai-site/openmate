# 共享依赖清单

## ripgrep (`rg`)

### 作用

1. Agent 能力模块中的 `grep` 工具基于 `rg` 做内容检索（支持正则）。
2. Agent 能力模块中的 `glob` 工具基于 `rg --files` 做文件名通配检索。

### 安装方式（Windows）

1. `winget install BurntSushi.ripgrep.MSVC`
2. `choco install ripgrep`
3. `scoop install ripgrep`

### 验证方式

1. 执行：`rg --version`
2. 若命令不可用，请确认 `rg.exe` 已加入 `PATH`。

### 备注

1. 当前项目对 `ripgrep` 的集成方式是“调用系统 `rg` 命令”，不是内嵌库。
2. 未安装 `rg` 时，`grep/glob` 工具会返回依赖缺失错误。

## Go 工具链

### 作用

1. 虚拟文件系统模块当前已切换为 Go 实现。
2. `vos` CLI 的开发、测试和构建依赖本机 Go 工具链。
3. Go 模块依赖明细见 [Go依赖.md](D:/XuKai/Project/vos/architecture/sharedInfo/Go依赖.md)。
4. Go 仓库结构与合并约定见 [Go仓库约定.md](D:/XuKai/Project/vos/architecture/sharedInfo/Go仓库约定.md)。

### 安装方式（Windows）

1. `winget install -e --id GoLang.Go`
2. 或使用 Go 官方安装包安装

### 验证方式

1. 执行：`go version`
2. 建议执行：`go env GOPROXY`

### 备注

1. 当前 VFS 测试在部分 Windows 环境下可能无法写系统默认 Go build cache，可通过设置仓库内 `GOCACHE` 规避。
2. 当前实现未依赖第三方 Go 库，标准工具链即可完成构建和测试。
