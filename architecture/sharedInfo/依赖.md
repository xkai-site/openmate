# 共享依赖清单

## Python 虚拟环境

### 作用

1. `openmate_agent` 继续使用 Python + Pydantic。
2. `openmate_pool` 的 Python adapter 继续由项目内 `.venv` 承载。
3. Python 测试统一在项目内虚拟环境执行。

### 约束

1. 使用仓库内 `.venv`。
2. 新增 Python 依赖时先更新 `requirements.txt`，再执行 `pip install -r requirements.txt`。
3. 不引入 conda、docker 作为 Python 环境管理手段。

## ripgrep (`rg`)

### 作用

1. Agent 能力模块中的 `grep` 工具基于 `rg` 做内容检索（支持正则）。
2. Agent 能力模块中的 `glob` 工具基于 `rg --files` 做文件名通配检索。

### 安装方式（Windows）

1. `winget install BurntSushi.ripgrep.MSVC`
2. `choco install ripgrep`
3. `scoop install ripgrep`

### 验证方式

1. 执行：`rg --version`
2. 若命令不可用，请确认 `rg.exe` 已加入 `PATH`

## Go 工具链

### 作用

1. `vos` 当前为 Go 实现。
2. `pool` 当前为 Go 运行时实现。
3. `schedule` 已确认采用 Go 技术栈推进。
4. Go 仓库结构约定见 `architecture/sharedInfo/Go仓库约定.md`
5. Go 依赖明细见 `architecture/sharedInfo/Go依赖.md`

### 安装方式（Windows）

1. `winget install -e --id GoLang.Go`
2. 或使用 Go 官方安装包安装

### 验证方式

1. 执行：`go version`
2. 建议执行：`go env GOPROXY`

### 备注

1. 部分 Windows 环境无法写系统默认 Go build cache，建议把 `GOCACHE` 和 `GOMODCACHE` 落到仓库内 `.openmate/`
2. 统一验证命令为：`go test ./...`

## CGO / C 编译器

### 作用

1. `pool` 当前依赖 `github.com/mattn/go-sqlite3`
2. 该依赖在本地构建时需要可用的 CGO / C 编译环境

### 验证方式

1. 执行：`go env CGO_ENABLED`
2. 执行：`gcc --version`
3. 若 `go test ./...` 报 `cgo` 或 `gcc` 缺失，再补装本机 C 编译器

### 备注

1. Windows 下可使用 MinGW-w64、MSYS2 gcc 或其他兼容 C 编译器
2. 这是 Go 构建依赖，不影响 Python 侧 `.venv` 约束
