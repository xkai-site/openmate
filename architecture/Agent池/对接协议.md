# Agent池对接协议

本文档面向会直接调用 `pool` 的模块。

当前基线：

1. `pool` 同时支持 OpenAI `Responses` 与 `Chat Completions` 两种请求形态。
2. 对外入口仍是 `CLI + JSON`。
3. `pool` 是网关和可观测层，不负责工具执行本身。
4. 两种协议的上下文都由调用方维护，`pool` 不维护会话历史。
5. 为保持前端调用稳定：当 `api_mode=chat_completions` 时，`pool` 仍支持接收 `request`（Responses 形态）并在池内自动转换为 Chat 请求。

## 1. 命令入口

Go 二进制：

```bash
openmate-pool --help
```

Python 包装：

```bash
python -m openmate_pool --help
```

当前命令：

1. `invoke`
2. `cap`
3. `records`
4. `usage`
5. `sync`

## 2. invoke 请求

### 2.1 外层请求结构

`pool invoke` 接收的 JSON 结构如下：

```json
{
  "request_id": "req-node-1",
  "node_id": "node-1",
  "request": {
    "input": "hello"
  },
  "chat_request": null,
  "timeout_ms": 30000,
  "route_policy": {
    "api_id": "openai-main"
  }
}
```

字段说明：

1. `request_id`
   调用侧自带的请求 ID。
2. `node_id`
   调用所属的 Node/任务单元。
3. `request`
   OpenAI `Responses create` 请求体。
4. `chat_request`
   OpenAI `Chat Completions` 请求体。
5. `timeout_ms`
   单次 provider 调用超时。
6. `route_policy.api_id`
   可选，指定命中某个 API 配置。

约束：

1. `request` 与 `chat_request` 必须二选一，不能同时传，也不能都不传。

### 2.2 request 字段规则

`request` 由上层传入，整体对齐 OpenAI `Responses create`。

当前建议使用的字段：

1. `input`
2. `instructions`
3. `tools`
4. `tool_choice`
5. `parallel_tool_calls`
6. `previous_response_id`
7. `max_output_tokens`
8. `temperature`
9. `top_p`
10. `metadata`
11. `reasoning`
12. `text`
13. `truncation`
14. `store`
15. `user`

当前限制：

1. 不允许上层传 `request.model`
   `model` 由 `model.json` 中命中的 API 配置注入。
2. `request.stream=true` 暂不支持。
3. `api_key/base_url/headers/request_defaults/pricing` 都不在请求里传，由 `model.json` 提供。

### 2.2.1 chat_request 字段规则

`chat_request` 由上层传入，整体对齐 OpenAI `Chat Completions create`。

当前建议使用的字段：

1. `messages`（必填）
2. `tools`
3. `tool_choice`
4. `response_format`
5. `max_tokens`
6. `temperature`
7. `top_p`
8. `user`
9. `store`

当前限制：

1. 不允许上层传 `chat_request.model`
   `model` 由 `model.json` 中命中的 API 配置注入。
2. `chat_request.stream=true` 暂不支持。
3. `pool` 不维护会话历史；多轮上下文由调用方自己维护并在每轮提交完整 `messages`。

兼容说明：

1. 若调用方继续只传 `request`（Responses 形态），并将 `api_mode` 切到 `chat_completions`，`pool` 会自动做协议转换，无需前端改入参结构。

### 2.3 最小文本调用示例

```json
{
  "request_id": "req-text-1",
  "node_id": "node-text-1",
  "request": {
    "input": "reply with pong"
  }
}
```

### 2.4 工具调用示例

首轮请求：

```json
{
  "request_id": "req-tool-1",
  "node_id": "node-tool-1",
  "request": {
    "input": "查询北京天气",
    "tools": [
      {
        "type": "function",
        "name": "get_weather",
        "description": "Get weather by city name.",
        "parameters": {
          "type": "object",
          "properties": {
            "city": {
              "type": "string"
            }
          },
          "required": [
            "city"
          ],
          "additionalProperties": false
        }
      }
    ],
    "tool_choice": "auto",
    "parallel_tool_calls": true
  }
}
```

如果模型返回 `function_call`，工具由上层执行。执行完后发下一轮：

```json
{
  "request_id": "req-tool-2",
  "node_id": "node-tool-1",
  "request": {
    "previous_response_id": "resp_123",
    "input": [
      {
        "type": "function_call_output",
        "call_id": "call_123",
        "output": "{\"city\":\"Beijing\",\"temperature_c\":23}"
      }
    ]
  }
}
```

### 2.5 Chat 工具调用示例

首轮请求（Chat）：

```json
{
  "request_id": "req-chat-tool-1",
  "node_id": "node-chat-tool-1",
  "chat_request": {
    "messages": [
      {
        "role": "user",
        "content": "查询北京天气"
      }
    ],
    "tools": [
      {
        "type": "function",
        "function": {
          "name": "get_weather",
          "description": "Get weather by city",
          "parameters": {
            "type": "object",
            "properties": {
              "city": {
                "type": "string"
              }
            },
            "required": [
              "city"
            ]
          }
        }
      }
    ],
    "tool_choice": "auto"
  }
}
```

如果模型返回 `tool_calls`，工具由上层执行并把结果以 `role=tool` 追加到下一轮 `messages` 中再调用。

## 3. invoke 返回

### 3.1 成功返回

成功时，CLI 返回码为 `0`，`stdout` 输出 JSON：

```json
{
  "invocation_id": "inv_123",
  "request_id": "req-text-1",
  "node_id": "node-text-1",
  "status": "success",
  "route": {
    "api_id": "openai-main",
    "provider": "openai_compatible",
    "model": "gpt-4.1"
  },
  "response": {
    "id": "resp_123",
    "object": "response",
    "model": "gpt-4.1",
    "status": "completed",
    "output": [
      {
        "type": "message",
        "role": "assistant",
        "status": "completed",
        "content": [
          {
            "type": "output_text",
            "text": "pong"
          }
        ]
      }
    ],
    "usage": {
      "input_tokens": 5,
      "input_tokens_details": {
        "cached_tokens": 0
      },
      "output_tokens": 3,
      "output_tokens_details": {
        "reasoning_tokens": 0
      },
      "total_tokens": 8
    }
  },
  "output_text": "pong",
  "usage": {
    "input_tokens": 5,
    "output_tokens": 3,
    "total_tokens": 8,
    "cached_input_tokens": 0,
    "reasoning_tokens": 0,
    "latency_ms": 420,
    "cost_usd": 0.00001
  },
  "timing": {
    "started_at": "2026-04-14T03:00:00Z",
    "finished_at": "2026-04-14T03:00:00.420Z",
    "latency_ms": 420
  },
  "error": null
}
```

字段说明：

1. `response`
   完整的 OpenAI `response` 对象。
2. `output_text`
   `pool` 从 `response.output` 中提取出的文本快捷字段。
3. `usage`
   网关标准化后的 usage 和成本信息。
4. `route`
   实际命中的 API/provider/model。
5. `timing`
   调用级时间信息。

### 3.2 调用失败返回

如果已经打到 provider，但最终失败，CLI 返回码为 `2`，`stdout` 仍输出结构化失败 JSON：

```json
{
  "invocation_id": "inv_124",
  "request_id": "req-text-2",
  "node_id": "node-text-2",
  "status": "failure",
  "route": {
    "api_id": "openai-main",
    "provider": "openai_compatible",
    "model": "gpt-4.1"
  },
  "response": null,
  "output_text": null,
  "usage": null,
  "timing": {
    "started_at": "2026-04-14T03:00:01Z",
    "finished_at": "2026-04-14T03:00:02Z",
    "latency_ms": 1000
  },
  "error": {
    "code": "provider_rate_limited",
    "message": "provider returned HTTP 429",
    "retryable": true,
    "provider_status_code": 429,
    "details": {
      "body": "{\"error\":{\"message\":\"rate limited\"}}"
    }
  }
}
```

### 3.3 非调用失败

如果是网关前置错误，例如：

1. `model.json` 缺失
2. 请求 JSON 非法
3. `request.model` 被错误传入
4. 无可用 API

则 CLI 返回码也是 `2`，但这类错误通常走 `stderr` 文本，而不是结构化 `InvokeResponse`。

## 7. model.json

### 7.1 示例

```json
{
  "global_max_concurrent": 20,
  "offline_failure_threshold": 3,
  "retry": {
    "max_attempts": 3,
    "base_backoff_ms": 200
  },
  "apis": [
    {
      "api_id": "openai-main",
      "provider": "openai_compatible",
      "model": "gpt-4.1",
      "base_url": "https://api.openai.com/v1",
      "api_key": "sk-***",
      "max_concurrent": 8,
      "enabled": true,
      "headers": {
        "OpenAI-Beta": "responses=v1"
      },
      "request_defaults": {
        "instructions": "You are a reliable assistant.",
        "temperature": 0.2,
        "parallel_tool_calls": true,
        "text": {
          "format": {
            "type": "text"
          }
        }
      },
      "pricing": {
        "input_per_1m_usd": 2.0,
        "output_per_1m_usd": 8.0,
        "cached_input_per_1m_usd": 0.5,
        "reasoning_per_1m_usd": 8.0
      }
    }
  ]
}
```

### 7.2 规则

1. `provider/model/base_url/api_key/max_concurrent/enabled` 是连接与路由必需字段。
2. `request_defaults` 是默认请求参数，请求显式传入的同名字段覆盖它。
3. `request_defaults` 里不允许出现 `model` 和 `input`。
4. `pricing` 用于成本估算，不影响真实 provider 计费。
5. `retry` 是网关级重试策略。

## 8. Python 调用

Python 侧仍然是同一份契约，只是通过适配器发 CLI：

```python
from openmate_pool.models import InvokeRequest, OpenAIResponsesRequest
from openmate_pool.pool import PoolGateway

gateway = PoolGateway()
response = gateway.invoke(
    InvokeRequest(
        request_id="req-1",
        node_id="node-1",
        request=OpenAIResponsesRequest(
            input="reply with pong"
        ),
    )
)
print(response.output_text)
```

Chat 请求示例：

```python
from openmate_pool.models import InvokeRequest, OpenAIChatCompletionsRequest
from openmate_pool.pool import PoolGateway

gateway = PoolGateway()
response = gateway.invoke(
    InvokeRequest(
        request_id="req-chat-1",
        node_id="node-chat-1",
        chat_request=OpenAIChatCompletionsRequest(
            messages=[{"role": "user", "content": "hello"}],
            tool_choice="auto",
        ),
    )
)
print(response.output_text)
```
